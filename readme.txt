Completely disables WordPress emoji scripts and styles on frontend and backend. No settings, no UI, zero overhead.

== Description ==

VB Disable Emojis is a lightweight performance plugin that completely removes emoji support from WordPress.

It disables all emoji-related scripts, styles and integrations across the entire site:
- frontend
- admin panel
- TinyMCE editor
- RSS feeds
- emails

There are no settings, no configuration and no database writes.
Activate the plugin to disable emojis. Deactivate it to restore default WordPress behavior.

Perfect for websites where emojis are not intentionally used and performance overhead should be minimized.

VB Disable Emojis is developed by VerBaste, a product brand of MATRO Digital Studio.

== Features ==

* Remove emoji scripts and styles
* Disable emojis in TinyMCE
* Disable emojis in RSS feeds and emails
* Remove emoji CDN resource hints
* Zero configuration
* Zero UI
* Zero database usage

== Installation ==

1. Upload the plugin folder to `/wp-content/plugins/` or install it via the WordPress plugin installer.
2. Activate the plugin.
3. Done. Emojis are disabled.

== Frequently Asked Questions ==

= Does this plugin add any settings or options? =

No. There are no settings pages, options or filters.

= Can I enable emojis only on frontend or backend? =

No. This plugin disables emojis globally by design.

= Will this break my site? =

No. WordPress works normally without emoji support.

= How do I re-enable emojis? =

Simply deactivate the plugin.
